# Enterprise Product Documentation & Architecture Blueprint
## Shelly Sharma – Professional Teacher Portfolio & Online English Academy

---

## 1. Executive Summary
This document serves as the master Product Requirement Document (PRD) and Design System specification for **Shelly Sharma – Professional Teacher Portfolio & Online English Academy**. It outlines a high-end, premium personal brand ecosystem designed to capture, display, and convert high-intent prospective students, educational institutions, and coaching clients. 

Unlike traditional flat teacher portfolios, this platform is engineered as a multi-tier conversion and personal branding funnel. It merges an authoritative educational pedigree (10+ years of institutional teaching, curriculum development, and authoring) with a highly approachable, warm, and elite private academy aesthetic. It is custom-tailored for the digital economy, optimized for search engines, and built on an accessible, high-performance architecture.

---

## 2. Product Vision
To establish Shelly Sharma as the preeminent global voice in English education and spoken language coaching—bridging the gap between formal academic mastery and modern conversational confidence. The platform aims to be the absolute gold standard of educational portfolios: a digital sanctum that commands trust, evokes warmth and elegance, and drives frictionless enrollment for her English speaking academy.

## 20. Motion Design System

### Animation Timings & Curves
*   `--transition-fast`: `200ms cubic-bezier(0.16, 1, 0.3, 1)` (Button hovers, input focus)
*   `--transition-smooth`: `400ms cubic-bezier(0.16, 1, 0.3, 1)` (Card transitions, tab switching, menu drop)
*   `--transition-slow`: `800ms cubic-bezier(0.16, 1, 0.3, 1)` (Hero entrances, section fade-ins)

### Animation Behaviors
*   **Hero Entrance:** The entire hero content transitions with a subtle upward movement and gradual fade-in, while the main portrait eases in smoothly.
*   **Section Entrances:** Staggered transitions reveal section content from the bottom up as the user scrolls, creating a natural reading rhythm.
*   **Interactive Cards:** Cards scale up gently with soft shadow changes when hovered, returning smoothly to their default state afterwards.
*   **Primary Buttons:** Hover actions expand button backgrounds and slide icons smoothly from left to right.
*   **Floating Elements:** Orbiting badges move continuously on a slow, rhythmic vertical path to create a sense of elegant depth.
*   **Timeline Connectors:** Scroll-driven progress indicators illuminate career markers chronologically as the viewport scrolls.
*   **Accessibility Controls:** Responsive elements respect user operating system rules (`@media (prefers-reduced-motion: reduce)`) by disabling complex motion and utilizing simple opacity transitions instead.

## 23. Accessibility Master Checklist (WCAG 2.2 AAA Target)
*   [ ] **Text Contrast:** Text elements must meet a minimum contrast ratio of `4.5:1` against their backgrounds (using rich charcoal and deep purples over soft off-whites).
*   [ ] **Keyboard Navigation:** Every interactive button, input field, and accordion link must be fully navigable using `Tab` and `Shift + Tab` keys.
*   [ ] **Screen Reader Compatibility:** Structural components utilize semantic tags (`<header>`, `<main>`, `<section>`, `<article>`, `<footer>`) with precise descriptive label attributes.
*   [ ] **Focus Indicators:** Interactive elements feature custom focus outlines to guarantee a consistent keyboard navigation experience.
*   [ ] **Motion Sensitivity:** Layout styles automatically scale down to simple fade transitions for users who prefer reduced motion.

## 24. Responsive Layout Strategy
*   **Desktop (`1280px` - `1920px`):** Generous whitespace layouts with balanced grids and large geometric hero frames.

### IX. Image SEO Strategy

To ensure all visual assets contribute to search performance, loading speeds, and accessibility, use the following rules for image management.

```
┌─────────────────────────────────────────────────────────────┐
│                 IMAGE METADATA & OPTIMIZATION               │
├─────────────────────────────────────────────────────────────┤
│  FILENAME: shelly-sharma-ma-english-teacher-pune.webp       │
│                                                             │
│  ALT TEXT: Shelly Sharma, certified PGT English teacher and  │
│  academy founder, holding an MA in English.                 │
│                                                             │
│  COMPRESSION: WebP Lossy (80% quality target)               │
└─────────────────────────────────────────────────────────────┘
```

#### 1. File Naming Rules
*   **No Generic Camera Names:** Never upload images with generic names like `IMG_0023.jpg` or `banner.png`.
*   **Descriptive Keywords:** Names must be clean, lowercase, and descriptive.
    *   *Primary Portrait Image:* `shelly-sharma-ma-english-teacher-pune.webp`
    *   *Course Banner Image:* `spoken-english-mastery-class-banner.webp`
    *   *Degree Certification:* `shelly-sharma-pune-university-bed-degree.webp`

#### 2. Dimensions & Image Formats
*   **Always WebP format:** All images must be converted to `.webp` or `.avif` to ensure optimized performance.
*   **Explicit Height and Width Attributes:** Images must define explicit height and width parameters in the HTML code to prevent layout shifts during page rendering.
*   **Hero Image Parameters:** Use high-resolution source images with precise crops, ensuring the portrait is cleanly centered against a transparent background.

#### 3. Image Metadata and Alt Tags
*   **Alt Tag Strategy:** Alternative text tags must clearly describe the image contents while naturally incorporating target keywords.
    *   *Primary Hero Image:* `alt="Shelly Sharma, certified PGT English teacher and academy founder, holding an MA in English."`
    *   *Timeline School Image:* `alt="Kaanger Valley Academy residential school campus in Raipur where Shelly Sharma taught English."`
*   **Image Title Tags:** Use descriptive title tags to provide additional contextual details for assistive technologies.

### X. Master Quality Assurance (QA) Checklist

Before compiling or deploying changes, verify the platform against this comprehensive quality control protocol.

#### 1. Search Engine Optimization (SEO)
- [ ] Ensure exactly one `<h1>` element exists on the page.
- [ ] Check that all external and anchor links feature descriptive descriptive link text.
- [ ] Verify that Google's Schema Markup Validator returns zero warnings or errors for JSON-LD schemas.
- [ ] Confirm all page configurations feature a self-referential canonical tag.
- [ ] Ensure alternative text (`alt`) is present on all active visual elements.

#### 2. Accessibility & Standards (WCAG 2.2 AAA Target)
- [ ] All page text combinations must meet or exceed a contrast ratio of `4.5:1`.
- [ ] Ensure all key page elements are fully navigable using the `Tab` and `Shift+Tab` keys.
- [ ] Interactive elements must display a high-contrast focus outline when selected.
- [ ] Screen readers must be able to read descriptive alternate tags for all icons and interactive items.
- [ ] Form elements must feature descriptive label elements.

#### 3. Core Page Performance
- [ ] Confirm that all main assets utilize next-generation WebP or AVIF formats.
- [ ] Ensure page fonts include `font-display: swap` parameters.
- [ ] Verify that visual layout shifts (CLS) are minimal during page loading.
- [ ] Confirm all production builds are compiled into clean, optimized bundle files.

#### 4. Responsive Design & Usability
- [ ] Check layouts across diverse screen sizes (including desktop, mobile, and small tablets).
- [ ] Confirm all mobile touch targets are at least `48px` in size to ensure comfortable navigation.
- [ ] Check that the navigation system transitions into a user-friendly floating mobile drawer on smaller screens.
- [ ] Confirm forms fit cleanly on narrow screens without introducing horizontal page scrolling.
- [ ] Ensure typography scales remain fully readable on compact displays.

#### 5. Form Submissions & Interactive Security
- [ ] Confirm input fields display clear, visible validation states when processing data.
- [ ] Ensure contact forms handle empty inputs gracefully without throwing internal errors.
- [ ] Verify success screens display responsive confirm animations after form submission.
- [ ] Check that all outbound links feature `noopener noreferrer` parameters to protect user security.
- [ ] Ensure the site runs over secure HTTPS connections.
